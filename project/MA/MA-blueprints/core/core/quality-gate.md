# Quality Gate — Check Before You Submit

Run through this checklist BEFORE writing your final summary. Every task must pass these checks.

## Completeness Check

1. Re-read the original request. Did you address every part of it?
2. Count how many things the user asked for. Did you deliver all of them?
3. If you wrote files, do they contain real content — not placeholders or TODOs?

## Accuracy Check

4. Did you state any facts? Verify each one came from a source (web search, memory, file) — not from your own generation.
5. If you used numbers or statistics, are they from the source material — not made up?
6. If you quoted something, is it accurate to the source?

## Output Check

7. Did you write outputs to workspace files (not dumped into chat)?
8. Are file names descriptive? (`research-notes.md` not `output.md`)
9. If you created multiple files, is there a clear naming pattern?

## Scope Check

10. Did you ONLY do what was asked? No extra features, no unrequested additions.
11. Did you stay within the task type? (Research task = research, not coding.)
12. Is your final summary brief? (2–4 sentences about what you did and where to find it.)

## Recovery Check

13. If any step failed, did you note the failure and explain what you tried?
14. If you could not complete something, did you say so clearly — not hide it?

## Quick Pass/Fail

Before submitting, answer YES or NO:
- Did I address the user's actual request? → must be YES
- Are all facts sourced, not invented? → must be YES
- Are outputs in workspace files? → must be YES for multi-sentence output
- Is my summary under 5 sentences? → must be YES

If any answer is NO, fix it before submitting.
