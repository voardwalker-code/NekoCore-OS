## [Unreleased]

### Changed
- **Book Ingest project folder organization** — Book Ingest UI now prompts for a "Project Name" before upload. MA Server creates a `projects/{slug}/` folder in the workspace and stores all book chunks inside it. The project folder path flows through URL params to MA's chat, where the task runner injects a mandatory file organization rule into both planning and step system prompts, ensuring all output files (character registries, memory JSONs, reports) go into the named project folder instead of being scattered at the workspace root. Backward-compatible `_findBookDir()` helper searches project folders first, then legacy `books/` path.
- **Direct book chunk tools** — Added `book_list_chunks` and `book_read_chunk` as workspace tools in MA, replacing the broken `web_fetch` approach (SSRF guard correctly blocks localhost HTTP calls). The LLM now reads book chunks directly from the filesystem instead of trying to HTTP-fetch from MA's own server. Updated `book_ingestion.md` blueprint to use the new tools exclusively.

### Changed
- **Entity creation no longer auto-checkouts** â€” All 5 entity creation endpoints (`postEntitiesCreate`, `postEntitiesCreateHatch`, `postEntitiesCreateGuided`, `postEntitiesCreateCharacter`, `postHatch`) no longer activate, load, or checkout the entity after creation. Entity files are written to disk and the desktop workspace is created, but the entity is not loaded into runtime or the chat UI. The user must manually select the entity from the sidebar to begin chatting. Client `syncParentAfterCreate()` now just refreshes the sidebar entity list and closes the creator window.
- **Entity naming format** â€” Entity folders changed from `entity_<hex>` to `Entity-<NameSlug>-<shortHex>` (e.g., `Entity-Luna-a1b2c3`). All entity creation paths (hatch, guided, character, client-side) use the new format. `getEntityRoot()` checks new format first, falls back to legacy `entity_` for backward compatibility.
- **App menu widened** â€” Launcher grid minimum column width increased from 200px to 240px, app icons from 30px to 34px, app name text given flex overflow so names are no longer clipped.

### Fixed
- **Creator progress spinner frozen on first step** â€” The entity creation progress overlay showed a spinning icon on "Generating identity and personality" but never advanced to subsequent steps while waiting for the server. Added a timed stepper (`startHatchStepTimer`) that advances the spinner through each step at regular intervals (4-5s). When the server responds, all steps are immediately marked complete. Steps reset properly between creations. CSS `.hatch-step.complete` now includes `opacity:1` to override inline opacity.
- **Settings per-stage model overrides not saving** â€” The 3 per-stage model override fields (Subconscious, Dream Engine, Orchestrator) in the Advanced settings section were read-only in practice: `simpleSaveConfig()` only saved the main provider but never sent the per-stage values to the server. Now each non-empty field is saved to `/api/entity-config` with auto-detected provider type and API key inheritance. Clearing a field removes the override so the aspect falls back to the main model.
- **Entity first-chat acts like continuation** â€” `getConsciousPrompt()` always injected a "MID-CONVERSATION RULE" telling the entity to never introduce itself, even on the very first message. Now checks `chatHistory.length`: if empty, injects a FIRST CONVERSATION RULE ("introduce yourself naturally") instead. Entities now greet new users with a short, warm introduction rather than pretending they already know them.
- **Visualizer entity dropdown still empty after creation** â€” `postEntitiesCreateHatch` and `postHatch` called `setActiveEntity()` but never `entityManager.loadEntity()`, so `/api/entities/current` returned `null` for the new entity. The Visualizerâ€™s fallback (inject current entity into dropdown) never fired. Added `loadEntity()` before `setActiveEntity()` in both endpoints.
- **Entity checkout bleed-through on create** â€” Entity creation endpoints now properly call `entityCheckout.releaseAllForAccount()` and `entityCheckout.checkout()` so created entities are immediately checked out for the user. Previously, creation set the server-side active entity but never updated the checkout system, causing stale entity context to persist in the chat pipeline.
- **Brain loop orphan on entity switch** â€” `_reinitBrainLoop()` now stops the old brain loop (saving state) before creating the new one. Previously, switching entities overwrote the reference without stopping, leaving orphaned timers running for the prior entity.
- **Client auto-checkout after entity creation** â€” `syncParentAfterCreate()` now calls `checkoutEntity()` instead of `sidebarSelectEntity()`, so the chat UI properly resets and the new entity is immediately usable.
- **Visualizer entity dropdown not showing new entities** â€” Added MutationObserver on the Visualizer window's `focused` class so the entity picker refreshes when the user switches to the Visualizer within the OS shell, not just on browser-level tab changes.
- **Client entity ID normalization** â€” `normalizeEntityId` in entity-ui.js now strips the `Entity-` prefix in addition to `entity_`, matching server-side normalization for the new entity folder format.
- **Visualizer works for non-active entities** â€” `/api/traces` and `/api/belief-graph/nodes` now create temporary TraceGraph/BeliefGraph from disk when the requested entity is not the active entity, instead of returning empty data.
- **Ollama extended thinking disabled** â€” Hard constraint in `resolveCapabilities()` forces `extendedThinking: false` and `adaptiveThinking: false` for Ollama providers regardless of profile-level overrides. Defense-in-depth stripping of thinking prompt suffix from Ollama system messages in `llm-interface.js`.
