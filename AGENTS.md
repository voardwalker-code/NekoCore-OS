# Agent Operating Rules

For this project, `WORKLOG.md` is the authoritative process document.

## Mandatory behavior

1. Reference `WORKLOG.md` in every message.
2. Validate planned actions against `WORKLOG.md` before editing code or docs.
3. Respect the active phase/slice and policy constraints documented in `WORKLOG.md`.
4. When uncertain, prefer `WORKLOG.md` over inferred workflow.

## Required line in each message
